## 2. 统计口径

ARMCC map 表中的字段：

```text
Code (inc. data)   RO Data   RW Data   ZI Data
```

本文档按以下口径计算：

```text
ROM = Code + RO Data + RW Data
RAM = RW Data + ZI Data
```

说明：

- `Code`：代码段，占用 ROM。
- `RO Data`：只读常量，占用 ROM。
- `RW Data`：有初值全局变量，运行时占 RAM，同时其初值也占 ROM。
- `ZI Data`：零初始化全局变量/BSS，只占 RAM。
- `Debug` 字段不进入固件镜像，不计入 ROM/RAM。

注意：

- 这是静态链接后的对象文件归类统计，不等于模块开关前后的真实增量。真实增量需要分别关闭模块后重新编译对比 map。
- FreeRTOS 任务栈通常从 FreeRTOS heap 动态分配，已经体现在 `heap_4.o` 的 64 KB 静态 heap 池中。
- LVGL 内部 heap 已体现在 `lv_mem_core_builtin.o` 的 48 KB 静态内存池中。
- LwIP 的 `MEM_SIZE`、`PBUF_POOL_SIZE`、`MEMP_*` 等池已经体现在 LwIP RAM 中。

## 6. 各模块说明

### 6.1 LVGL

当前消耗：

```text
ROM 215168 bytes (210.13 KB)
RAM  56132 bytes (54.82 KB)
```

主要组成：

- LVGL core、object、style、display、refresh。
- 软件绘制后端。
- RGB565/RGB565 swapped 等多个颜色格式混合函数。
- `lv_label.c`。
- `lv_arc.c`。
- Montserrat 12/14/16 字体。
- 48 KB LVGL 内部 heap。

当前资源环页面实际使用：

- `lv_label`
- `lv_arc`
- `lv_font_montserrat_12`
- `lv_font_montserrat_14`
- `LV_COLOR_FORMAT_RGB565_SWAPPED`

优化方向：

- 继续裁剪 `lv_conf.h` 中未使用的控件和库。
- 如果确认只使用 RGB565 swapped，可尝试裁剪无关 draw blend 格式源文件。
- 关闭默认主题或只保留 simple theme。
- 减小 `LV_MEM_SIZE`，例如从 48 KB 调到 24 KB 或 32 KB 后做压力测试。
- 移除未使用字体，例如不使用 16 号字体时关闭 `LV_FONT_MONTSERRAT_16`。

### 6.2 FreeRTOS

当前消耗：

```text
ROM 6114 bytes (5.97 KB)
RAM 65996 bytes (64.45 KB)
```

RAM 主要来自：

```text
heap_4.o 65568 bytes
```

说明：

- FreeRTOS 内核代码本身 ROM/RAM 都不大。
- RAM 大头是 `configTOTAL_HEAP_SIZE`。
- 任务栈、队列、信号量、定时器等动态对象都从 FreeRTOS heap 分配。

优化方向：

- 根据任务栈高水位调小各任务 stack。
- 根据运行情况调小 `configTOTAL_HEAP_SIZE`。
- 关闭未使用 FreeRTOS 特性，例如 timers/event groups/stream buffer 需要结合代码确认。

### 6.3 LwIP

当前消耗：

```text
ROM 22746 bytes (22.21 KB)
RAM 45271 bytes (44.21 KB)
```

RAM 主要来自：

```text
memp.o 26898 bytes
mem.o  16415 bytes
```

说明：

- 当前 RNDIS 使用 LwIP raw/no-OS 模式。
- `MEM_SIZE`、`PBUF_POOL_SIZE`、`MEMP_NUM_*` 会直接影响 RAM。
- 目前 TCP 关闭，主要保留 IPv4、ARP、ICMP、UDP。

优化方向：

- 减小 `MEM_SIZE`。
- 减小 `PBUF_POOL_SIZE` 和 `PBUF_POOL_BUFSIZE`。
- 继续关闭未使用协议。
- 如果只做 ping/基础 UDP，可进一步裁剪。

### 6.4 USB Device/RNDIS

当前消耗：

```text
ROM 12997 bytes (12.69 KB)
RAM  7546 bytes (7.37 KB)
```

主要组成：

- USB Device core。
- CDC/RNDIS class。
- RNDIS interface 缓冲区。
- USB PCD/LL USB 部分在本文归入 USB Device/RNDIS 或 HAL，具体边界按对象文件名划分。

优化方向：

- 检查 RNDIS RX/TX buffer 大小。
- 如果不使用 USB CDC，可保持 CDC class 不编译。
- RNDIS 和 LwIP 需要一起评估，单独裁剪 USB 往往收益有限。

### 6.5 LittleFS/VFS

当前消耗：

```text
ROM 25821 bytes (25.22 KB)
RAM  1940 bytes (1.89 KB)
```

主要组成：

- `lfs.o` 是 ROM 最大来源，约 20.6 KB。
- `littlefs_sd.o` 负责 SD 块设备适配和测试缓冲。
- `fs.o`、`vfs.o` 是工程内文件系统封装。

优化方向：

- 如果只保留基础文件读写，可以减少 VFS/shell 暴露的功能。
- 检查 `littlefs_sd.o` 中 raw self test 缓冲是否必须常驻。
- 若只需要 FATFS 或裸 SD，可以替换 littlefs，但需要重新评估可靠性。

## 7. 运行时额外资源

静态 map 无法直接体现所有运行时峰值，需要关注：

- 每个 FreeRTOS 任务实际 stack 使用量。
- LVGL 动态对象分配量。
- littlefs 文件打开时的临时对象。
- USB/LwIP 数据包瞬时排队。
- printf/sprintf 调用栈开销。

建议在运行时周期打印：

```c
xPortGetFreeHeapSize()
xPortGetMinimumEverFreeHeapSize()
uxTaskGetStackHighWaterMark(task)
```

LVGL 可关注：

```c
lv_mem_monitor_t mon;
lv_mem_monitor(&mon);
```

如果开启 LVGL mem monitor，需要确认当前 LVGL 版本和配置中相关 API 可用。

## 8. 优化优先级

按收益排序：

1. LVGL ROM 裁剪：减少未用 draw 格式、字体、主题、控件。
2. FreeRTOS RAM：根据任务栈高水位调小 `configTOTAL_HEAP_SIZE` 和任务 stack。
3. LwIP RAM：调小 `MEM_SIZE`、`PBUF_POOL_SIZE`、`MEMP_NUM_*`。
4. LVGL RAM：调小 `LV_MEM_SIZE`，观察资源页和后续 UI 是否稳定。
5. LittleFS ROM：如果文件系统功能简单，可评估更小的封装或禁用部分 shell 文件命令。
6. HAL ROM：移除未使用外设 HAL 源文件。

## 9. 重新生成本文数据的方法

构建：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\build.ps1
```

查看总量：

```powershell
Select-String -Path H7Lttit\MDK-ARM\H7Lttit\H7Lttit.map `
  -Pattern "Total RO|Total RW|Total ROM|Grand Totals"
```

对象级统计位置：

```text
Image component sizes
Code (inc. data)   RO Data   RW Data   ZI Data   Debug   Object Name
```

模块统计需要把对象文件按模块归类。当前文档使用的归类规则是：

```text
lv_*                         -> LVGL
tasks/queue/list/heap_4/port -> FreeRTOS
etharp/ip4/mem/memp/pbuf     -> LwIP
usbd_*/usb_device/ll_usb     -> USB Device/RNDIS
lfs/littlefs/fs/vfs          -> LittleFS/VFS
lcd/st7735                   -> BSP LCD/ST7735
stm32h7xx_hal/gpio/spi/...   -> STM32 HAL/Startup/Drivers
shell/comm/vim/lib           -> Shell/VIM/Lib
main.o                       -> Application main
qspi/w25q                    -> QSPI/W25Q
```

如果模块拆分或文件名变化，需要同步更新归类规则。
